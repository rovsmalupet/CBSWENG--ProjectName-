## Deployment to Railway

### Step 1: Push Changes to GitHub

```bash
git add .
git commit -m "Prepare backend for production deployment"
git push origin main
```

**⚠️ CRITICAL:** Make sure `.env` is NOT pushed (check with `git log --all -p backend/.env`)

### Step 2: Create Railway Account

1. Go to [railway.app](https://railway.app)
2. Sign up with GitHub
3. Click **"Create New Project"**
4. Select **"Deploy from GitHub Repo"**

### Step 3: Connect Repository

1. Select your GitHub repo: `CBSWENG--ProjectName-`
2. Railway will auto-detect it's Node.js
3. It will ask for root path: **Enter `backend`** (if not auto-detected)

### Step 4: Add PostgreSQL Database

1. In Railway dashboard, click **"+ Add Service"**
2. Select **"PostgreSQL"**
3. Railway auto-generates `DATABASE_URL` and `DIRECT_URL`

### Step 5: Configure Environment Variables

In Railway, go to **Variables** and set:

```
NODE_ENV=production
PORT=3000
FRONTEND_URL=https://your-vercel-frontend-url.com
# DATABASE_URL and DIRECT_URL are auto-generated by Railway
```

### Step 6: Configure Build Commands (if needed)

In Railway **Settings**, set:

- **Build Command:** `npm install && npx prisma generate`
- **Start Command:** `npm start`

### Step 7: Deploy & Monitor

1. Railway will auto-deploy on push to `main`
2. Your backend URL appears in Railway dashboard
3. Test it: `https://your-railway-app.up.railway.app/health`

## Important Notes for Railway Deployment

### Database Migrations

- Railway automatically runs `postinstall` script
- Your Prisma migrations are included in the repo
- First deployment will create database schema automatically

### Environment Variables in Production

Never commit `.env`! Railway uses environment variables instead:

- Set all variables in Railway dashboard
- Railway securely manages them

### CORS Configuration

The backend now supports:

- Local development: `http://localhost:3000`, `http://localhost:5173`
- Production: Set `FRONTEND_URL` in Railway variables
- Update when you deploy frontend to Vercel

## Testing Before Deployment

### Test locally first:

```bash
cd backend
npm install
npm start
# Test: curl http://localhost:3000/health
```

### Verify environment variables:

- `.env` has placeholder values for local dev
- `.env.example` documents all required variables
- Never commit real credentials

## Frontend Integration

After backend is deployed:

1. Get your Railway backend URL
2. Set `FRONTEND_URL` in your React fetch calls for production
3. Deploy frontend to Vercel
4. Set `VITE_API_URL` or similar env variable to your Railway backend URL

## Troubleshooting

### "Migration failed" error

- Check `DATABASE_URL` and `DIRECT_URL` are set in Railway
- Ensure database is in `DIRECT_URL`

### CORS errors from frontend

- Add your frontend's Vercel URL to `FRONTEND_URL` in Railway variables
- Restart deployment after updating

### "Cannot find module" errors

- Ensure `npm install` ran successfully
- Check `postinstall` script executed

### 500 errors

- Check Railway logs: Click **"View Logs"** in dashboard
- Look for database connection issues
